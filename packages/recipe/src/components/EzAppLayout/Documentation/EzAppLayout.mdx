# EzAppLayout

The application layout component provides common ways to arrange the content of a screen by controling the horizontal margins of the content area. The application layout wraps the main page content and houses the primary navigation for the application.

<Primary />

## Props

<Controls of={DefaultStories.Default} sort="requiredFirst" />

## Best practices

Application layouts should:

- Only appear once per screen
- Host the page content
- Optionally wrap the page content with a navigation component, where multiple top-level pages are needed

## Structure of a screen

A screen is the entire user interface of an application.

A typical screen in Recipe consists of several layers. The application layout is the outermost layer. Typically, the application layout will contain a navigation component (`EzNavigation`), and inside that a page header component (`EzPageHeader`) followed by a page component (`EzPage`).

The application layout component ensures that content containers are arranged in an appropriate way across all screen sizes.

## Full width layout

This is the default and should be used for pages where the user benefits from the content taking up the available screen width, like on a page with a Table.

Use the full width layout for wide list or table content that may benefit from more horizontal space.

## Centered layout

Use the centered layout to constrain page width to improve readability or focus attention on the primary page content. Pages with text-heavy cards or forms typically benefit from being constrained to a max-width.
